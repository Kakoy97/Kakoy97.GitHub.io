# 三年，三个项目，一个轮回

时隔三年重新拾起博客，翻看以前的文字，那种“真切想要变好”的劲头依然打动我。

这三年在 **jzyx**（前公司），我确实学到了不少。从性格的打磨到思维的转变，我甚至觉得有些幸运——因为我接触了三个完全处于不同生命周期的项目。

从**立项期的混沌**，到**研发期的冲刺**，再到**运营期的维护**，我也算完整的经历了游戏的一生🤣。今天想跳出具体的技术细节，聊聊这三段经历带给我的思考。

## 01. 立项期：为“单打独斗”买单

第一个项目刚立项不久，团队里充斥着多巴胺。大家干劲十足，想法满天飞，业务迭代快到每个月都像是在做一款新游戏。

因为处于早期，我有机会接触到底层模块。我开发的第一个模块是**资源管理系统**。现在回过头看，设计简直是**一坨**。值得夸赞的，是我当时为了方便测试同学，顺手做了一个可视化的测试看板。

但在那个阶段，我犯了新人最容易犯的错：**好面子，喜欢逞强**。

那时候我觉得：“我自己一个人能做好。如果请求别人帮助，就是我能力不行。”
即使业务有风险，我也宁可独自死磕加班，想思路想到头疼也不愿开口。

**结果是惨痛的：**
这种**“闷头蛮干”**不仅没让我证明自己，反而让我写出了不少 Bug。当时配合的一位测试伙伴给我的评价直接就是：“能力不太行”。

> **挨打要立正。**

在导师的开导下，我才意识到：**拒绝外部的帮助，本质上是拒绝了团队的协作杠杆。** 这种闭门造车的思维，不仅阻碍了项目，更阻碍了我的成长。

## 02. 研发期：工业化与协作的洗礼

第二个项目是一款开发中期的卡牌游戏。这对我来说是一次真正的“正规军”洗礼。

虽然是异地协同（团队在另一个城市的子公司），但我在这里学到了真正的**工程化思维**。团队非常重视效率和上下游配合，整体框架吸取了公司以往的痛点，定期清除死代码（Dead Code），防止冗余。

最让我眼前一亮的是他们的工具链：
* **PSD 转预制体 (PSD2UGUI)**：极大解放了拼 UI 的生产力。
* **多语言转换 (I18n Tool)**：自动化的 Key-Value 映射。
* **UI 状态控制器**：将复杂的 UI 逻辑状态化。

这是一个典型的“小步快跑”项目，每周一个版本，排期压得很紧。我们三人小组每天都会同步进度（Stand-up meeting），核心原则只有一条：**暴露风险**。

有风险？要么大家匀一匀任务，要么立刻向上请求支援。

慢慢地，我开始主导一些模块。从根据策划案评估排期，到拆解任务、分配工时。虽然加班依然严重，但我学会了**任务评估**和**架构设计**。

在团队一年半的努力下，项目成功上线，成绩也很不错。这段经历让我明白：**依靠体系和工具的力量，远胜过依赖个人的爆发力。**

## 03. 运营期：在“屎山”上雕花与重构

第三个项目是一款上线两三年的老牌卡牌手游。

做过长线运营的同学都知道，这类项目都有一个通病：**架构老旧，屎山堆积**。单单一个活动框架的逻辑就绕得人眼花缭乱。刚接手时，我忍不住吐槽：“查一个活动 Bug 简直要吐了。”

经历过第二个项目的现代化流程，再看这里，简直是从现代回到了 90 年代：
* UI 还原严格且痛苦，全部需要手动拼（没有 PSD 工具）。
* 美术人手不足，工具推动缓慢。

最让我痛苦的是**养成系统**。项目里存在大量结构类似的养成“胚子”，有些仅有细微的独立属性差别。就因为这点差别，前人整了 N 套数据结构，写了 N 个几乎一样的详情界面。

**维护这种代码，是一种精神折磨。**

后来我一不做二不休，直接找上级申请：**重构掉所有养成详情界面。**

我的思路是**“节点化” (Node-based)**：
1.  把每一块信息抽离成独立的 UI 节点。
2.  把养成数据包装成统一的 Adapter。
3.  由数据自己决定“我要组合哪些节点”。

**效果是立竿见影的：**
原本开发一个新养成界面需要半天，重构后缩短到了 **30 分钟**。即使有特殊需求，也只需要扩展一个特定节点即可。

趁着业务节奏减缓，我也顺势推动了组件化和效率工具的引入。在旧项目里做重构虽然痛苦，但这种“在屎山中开辟净土”的成就感，确实是独一份的。

## 04. 写在最后

这三个项目，不仅是技术的磨练，更是心态的蜕变。我不再是那个只听策划需求、好面子的“代码工具人”，而是开始思考架构、效率和团队价值。

**2025 年初，我选择了离开。**

说实话，这其中有很大一部分原因是对 **AI 崛起**的焦虑。看着 GPT 和各种 Copilot 的进化，我常在想：纯粹的执行者还有多少生存空间？

另一方面，我也想看看自己能不能做点什么，为所谓的“35 岁危机”提前探探路。

我也许不再是一个大厂的“螺丝钉”，但我依然是一个热爱创造的开发者。
